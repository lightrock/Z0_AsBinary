# Leibniz, Binary, Yin/Yang, and Process Receipts

## Status

This is a sober Leibniz note for the Maxwell-completion / closure-machine program.

It does not claim that Leibniz got binary arithmetic from the I Ching. The
historical relationship is more careful: Leibniz had developed binary arithmetic
before the famous I Ching correspondence, and later recognized a formal analogy
between his 0/1 arithmetic and the yin/yang line structure of the hexagrams.

The useful contribution is not an origin myth.

The useful contribution is:

```text
binary difference can be a generative grammar
0/1 is not merely a number notation
broken/unbroken, yin/yang, off/on can encode state, polarity, alternation, and transformation
```

Through the process-first lens, the important phrase is:

```text
yin/yang is not a static symbol pair; it is a running alternation grammar
```

---

## Historical anchor

Leibniz developed binary arithmetic using `0` and `1`, including conversion and
arithmetic rules. He later saw the I Ching hexagrams as a striking formal match:
solid and broken lines could be read as binary distinctions.

The careful statement is:

```text
Leibniz did not need the I Ching to invent binary arithmetic.
The I Ching gave Leibniz a deep symbolic confirmation that binary opposition and
combination could be a world-grammar.
```

That distinction matters. We want the receipt, not the myth.

## What Leibniz adds

Leibniz gives a pre-digital lesson:

```text
from two marks, generate many forms
from 0 and 1, build number
from absence and presence, build symbolic universe
from distinction, build computation-like structure
```

The useful contribution is not only base-2 notation. It is the recognition that a
minimal distinction can generate a large ordered universe of forms.

In repo language:

```text
one bit is not only payload
one bit can be a selector, trigger, polarity, boundary, or admissibility mark
```

That plugs directly into the Shannon correction in the BBQ lineup.

## What yin/yang adds

Yin/yang is often treated as a decorative dualism. Through this project lens,
that misses the point.

The useful version is:

```text
yin/yang is a dynamic complementary pair
the pair is not finished until it changes, alternates, balances, or transforms
hexagrams are not merely six stored bits; they are structured change states
```

A static binary string says:

```text
state
```

A yin/yang process grammar says:

```text
state -> tendency -> change -> new state -> interpretation / receipt
```

That is much closer to the closure-machine view.

## Machine receipt

Through the closure-machine lens, Leibniz + yin/yang gives this receipt:

```text
minimal opposition can generate a state space
state space can be ordered by transformation
transformation can be read as process, not just representation
```

That makes binary relevant before modern computers:

```text
binary is not only storage
binary can be an executable distinction
```

This matters for `Z0_AsBinary` because the repo does not merely ask whether the
published `Z0` digits can be converted into bits. It asks whether those bits do
anything under a transformation rule.

So the lineage is:

```text
Leibniz: 0/1 can generate an ordered universe of number.
I Ching / yin-yang: binary opposition can be interpreted as change-process.
Shannon: bits can be measured cleanly as information.
Z0_AsBinary: a bit pattern can be run as a closure candidate.
```


```text
Leibniz and the I Ching/Fu Xi hexagram tradition preserve a deep formal receipt:
minimal complementary distinction can generate structured worlds of state and
change. The Maxwell-completion program asks what physical distinction actually
runs, closes, and leaves electromagnetic receipts.
```
